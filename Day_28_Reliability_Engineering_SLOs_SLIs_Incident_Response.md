# Day 28: Reliability Engineering — SLOs, SLIs, and Incident Response

## Learning Objectives
- Define SLIs (latency, error rate), SLOs, and error budgets
- Incident response: runbooks, postmortems, and blameless culture
- Chaos engineering basics

## Why This Matters
Reliability practices ensure systems meet user expectations and teams can learn from failures.

---

## Hands-On Exercise
- Define SLIs and SLOs for a sample service and create simple alerts based on thresholds.

## Mini Project
- Run a small chaos experiment (stop a replica) and observe system behavior; update runbook accordingly.

## Expected Outcome
You can set measurable reliability goals, monitor them, and run basic incident response.
