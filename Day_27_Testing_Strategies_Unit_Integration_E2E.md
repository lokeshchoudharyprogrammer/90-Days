# Day 27: Testing Strategies — Unit, Integration, and End-to-End

## Learning Objectives
- Test pyramid: unit tests, integration tests, e2e tests
- Mocking vs real dependencies, test data management
- Contract testing for services (PACT)

## Why This Matters
Proper testing reduces regressions and provides confidence for safe deployments.

---

## Hands-On Exercise
- Write unit tests for core modules and integration tests that run against a test DB or test container.

## Mini Project
- Implement contract tests between two services and run them in CI.

## Expected Outcome
You can design a balanced test suite and integrate tests into CI pipelines.
