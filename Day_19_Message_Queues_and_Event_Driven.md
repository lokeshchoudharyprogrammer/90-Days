# Day 19: Message Queues & Event-Driven Architectures

## Learning Objectives
- Understand pub/sub, message queues, and durable messaging
- Learn systems: RabbitMQ, Kafka, SQS
- Design patterns: event sourcing, CQRS, at-least-once vs exactly-once semantics

## Why This Matters
Asynchronous messaging decouples services and improves resilience and throughput.

---

## Hands-On Exercise
- Implement a producer/consumer using RabbitMQ or Redis Streams; demonstrate ack/nack behavior.

## Mini Project
- Build an event-driven workflow: orders -> payments -> inventory adjustments via messaging.

## Expected Outcome
You can design and implement message-driven interactions and reason about delivery semantics.
